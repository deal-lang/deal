<objective>
Implement the SysML v2 codegen pipeline end-to-end: Rust emitter that walks the IR JSON, an offline JSON Schema validator using `jsonschema` 0.46 with a custom `Retrieve` impl against the bundled OMG schemas, and 8 hand-written golden fixtures that prove the byte-exact contract (REQ #6 acceptance gates: byte-equal + schema-valid).

Purpose: REQ-phase-2-3 + 3a + 3b together prove `deal build --target sysml-v2` produces importable SysML v2 JSON. The 8 fixtures cover the 5 mapping categories (part def, port usage, specialization, attribute usage, requirement def, traceability) PLUS the .dealx composition path. Plan 02-06's viewer-import smoke uses the showcase output produced by this plan's emitter.

Output: Two new Rust modules (sysml_v2.rs + schema_registry.rs); `deal build --target sysml-v2 [--validate]` subcommand wired through main.rs; 8 paired (.deal/.dealx + .expected.json) golden fixtures with snapshot tests; a dedicated `golden_fixture_schema_validity` cargo test that validates each hand-authored `.expected.json` against the bundled schema INDEPENDENT of the byte-match check (so a typo in a hand-authored fixture cannot silently anchor itself as the reference via insta accept); a README explaining the `cargo insta review` workflow per Pitfall 6.
</objective>
