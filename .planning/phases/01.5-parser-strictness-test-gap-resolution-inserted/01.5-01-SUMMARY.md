# Phase 01.5 Plan 01: Test Assertions & Property Span-Containment Summary

Strengthened 9 recovery test cases from vacuous `.len >= 0` / `.len >= 1` presence checks into the two-axis BDD contract from `parser_dealx.tag_balance` (exact diagnostic code + named AST shape), and added a new `property.span_containment` test that walks every AST node of the 19 showcase files plus 8 synthetic fixtures, locking the child-span-inside-parent-span invariant against snapshot-byte-equality blind spots.

## What Was Built

### Task 1 — Strengthen 9 recovery cases (commit `ab8f807`)

Replaced every `try std.testing.expect(... .len >= 0)` and `.len >= 1`-style assertion in the three recovery test files with the two-axis pattern mirrored from `parser_dealx_tag_balance.zig`:

**Axis A (diagnostic):** exact count by code via `Codes.e_*` constants by name. All assertions reference `diagnostics_mod.Codes` — no inline `"E0..."` string literals.

**Axis B (AST shape):** named structural property on the recovered AST at the expected position — definition name, member kind/name, root_tag count + name.

### Task 2 — property.span_containment (commit `2ed9f07`)

New test file `tests/unit/property_span_containment.zig` (469 lines) declaring `test "property.span_containment"`. Walks every AST node of:

- **15 .deal showcase files** (mirrors `parser_deal.coverage` corpus): vehicle/{battery, motor, behaviors, components, index}, interfaces/{electrical, thermal, connections, index}, requirements/{system, needs, index}, use-cases/{driving, charging, index}.
- **4 .dealx showcase files**: model/{traceability, vehicle}.dealx, model/variants/{performance, sedan}.dealx.
- **8 synthetic .deal fixtures** copied verbatim from `parser_deal_coverage.zig`: `visibility_wrapper`, `redefinition`, `real_literal_and_boolean`, `unary_expression`, `verification_block`, `precondition_postcondition`, `missing_def_kinds`, `requirement_usage`.

For every parent → child edge, asserts `child.span.start >= parent.span.start && child.span.end <= parent.span.end`. On violation, `std.debug.print` emits `NodeKind` + child span + parent span before returning `error.TestSpanContainmentViolation`.

The walker's `switch (node.payload)` is **exhaustive** — Zig 0.16.0's `union(NodeKind)` makes any missing arm a compile error. That exhaustiveness is part of the contract: any new NodeKind added in Plan 02 must add an arm and recurse into its children before the test compiles.

## Decisions Made

1. **statement.A — kept as well-formed baseline (option i, not option ii).** The plan offered two options: (i) keep as well-formed baseline and rename, or (ii) edit source to truly omit a semicolon and assert E0100. The empirical probe found that the proposed option-ii source `attribute a : T attribute b : T;` ALSO parses cleanly with **0 diagnostics and 2 attribute_usage members** — the parser is whitespace-tolerant at attribute boundaries. Option (i) is therefore the only honest contract; locked in as the well-formed baseline with strict shape assertions.

2. **statement.C — locked severe recovery (`members.len == 0`).** The plan predicted recovery would retain at least the `y` assignment. Empirically, the statement-sync sweep consumes through the next `;` (at end of `y = 7;`), dropping BOTH operator-statements. The exact-zero `members.len` assertion is what the plan's empirical-truth caveat instructs: lock the current behavior; Plan 02 strictness work will flag this gate if recovery improves.

3. **definition.C — locked `Broken` absorbs `part def Good` as `part_usage("def")`.** The plan predicted a separate `Good` part_def would emerge from recovery. Empirically the parser glues `part def Good` into Broken's body as a `part_usage` named `"def"` (because the `part` token, in modifier context inside a body, becomes part of a usage). Locked as the empirical truth with assertions checking both `attribute_usage("x")` AND `part_usage("def")` are present in `Broken.members`. Plan 02 strictness work that introduces definition-keyword recovery inside open bodies will flag this gate.

4. **BDD helpers duplicated, not extracted.** `countCode` + `firstWithCode` are 9 lines total, duplicated across only 3 test files. Extraction to `_diag_helpers.zig` would add a module dependency for marginal payoff; inlining keeps each test file self-contained.

5. **Grep gate phrasing in doc-comments.** Initial draft of `recovery_statement.zig`'s doc-comment included the literal phrase `\`.len >= 0\`` to reference the historic vacuous assertion, which itself triggered the `grep -nE 'len >= 0'` gate. Reworded to `\`.len\`-vacuous historic assertion` to satisfy the gate without losing the explanatory context.

## Deviations from Plan

### Empirical-truth deviations (per plan's caveat — assert reality, do not relax)

**1. [statement.A option choice] Plan recommended option (ii); empirical truth required option (i)**
- **Found during:** Task 1 probe
- **Plan prediction:** Option (ii) — `attribute a : T attribute b : T;` would emit `exactly one E0100 (e_expected_token)` with span between `T` and the second `attribute`, and `members.len >= 1`.
- **Empirical truth:** The proposed source emits **zero diagnostics** and `members.len == 2` (named `a` and `b`). The parser treats whitespace as a valid attribute-statement boundary in this position.
- **Resolution:** Switched to option (i) — kept the original well-formed source as the **well-formed baseline** asserting `0 diagnostics, 2 named members`. The case is now "well-formed two-attribute body" rather than "missing semicolon between attribute members" (which the parser would not detect as missing anyway). Documented in the file's top-level doc-comment.
- **Files modified:** tests/unit/recovery_statement.zig
- **Commit:** ab8f807

**2. [statement.C recovery shape] Plan predicted "y" assignment would survive recovery; empirical truth: both operator-statements dropped**
- **Found during:** Task 1 probe
- **Plan prediction:** Assert that the part_def has at least one member whose payload name field equals "y".
- **Empirical truth:** The statement-sync sweep at the missing `;` boundary consumes through to the next `;` (at end of `y = 7;`), dropping BOTH operator-statements. Recovered `part_def P` has `members.len == 0`.
- **Resolution:** Per plan's empirical-truth caveat ("DO NOT relax the assertion — instead... assert the empirical truth"), locked `members.len == 0` and documented in file doc-comment as "severe recovery shape; Plan 02 strictness will flag this gate if it changes".
- **Files modified:** tests/unit/recovery_statement.zig
- **Commit:** ab8f807

**3. [definition.C recovery shape] Plan predicted a separate `Good` part_def; empirical truth: absorbed as `part_usage("def")` inside Broken**
- **Found during:** Task 1 probe
- **Plan prediction:** Loop-find a part_def named "Good" in `df.definitions`.
- **Empirical truth:** Parser produces `definitions.len == 1` with the single part_def named `Broken` containing members `[attribute_usage("x"), part_usage("def")]`. The `part def Good` tokens are consumed as a usage member (the `part` keyword is consumed as a modifier inside an open body context).
- **Resolution:** Locked the empirical truth — assert exactly 1 part_def named `Broken`, and loop-find both `attribute_usage("x")` AND `part_usage("def")` in its members. Documented in file doc-comment.
- **Files modified:** tests/unit/recovery_definition.zig
- **Commit:** ab8f807
