## Deviations

Two minor cosmetic / safety adjustments (both reasonable, both surfaced by the executor):

1. **Per-invocation file:// protocol allowance during `git submodule add`.** Modern git denies `file://` clones by default (CVE-2022-39253 mitigation). The executor passed `-c protocol.file.allow=always` ONLY to the single `git submodule add` command — NOT to `~/.gitconfig`, `.git/config`, or `.gitmodules`. The committed URL is the canonical SSH form, so fresh-clone CI runners use SSH/GitHub and never touch the file-protocol allowance.
2. **Removed `IFS=$'\n\t'` from verify-fresh-worktree.sh after semgrep CWE-20 warning.** All variable expansions in the script are quoted; word-splitting behavior is irrelevant. Replaced with a justifying comment.

Neither deviation altered committed-state semantics; both are documented in this SUMMARY.
