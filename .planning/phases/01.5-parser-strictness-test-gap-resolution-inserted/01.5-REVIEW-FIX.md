## Skipped Issues

### IN-05: property.span_containment warns but does not fail on showcase diagnostics
**File:** `tests/unit/property_span_containment.zig:399-408` and `429-433`
**Reason:** Intentional design per the file's own top-of-file doc-comment ("The property test only asserts span containment ‚Äî log to stderr but do not fail"). The review itself classifies this as INFO with `**Fix:** None required` ‚Äî the existing coverage by `parser_deal.coverage` and `parser_deal.snapshot` already enforces well-formedness on showcase files, so the warn-but-do-not-fail behavior here is defense-in-depth weakening (not a contract gap). Promoting the warning to a hard failure would provide a third independent gate but is a discretionary enhancement, not a correctness fix.
**Original issue:** Showcase files are well-formed by Phase 1 contract; any diagnostic emitted while parsing them is a regression. The property test logs (`std.debug.print`) when `diags.items.len > 0` but does NOT fail. A regression in well-formed parsing that surfaces as a diagnostic on a showcase file would slip past this test ‚Äî the property test would still pass as long as spans are contained.
