### Remediation

Plan `01.5-05-tests-showcase-submodule-and-verify-harden-PLAN.md` (this plan)
committed the following:

- spec/ added as a git submodule of deal/ at mount path `spec`, pinned to
  spec/'s HEAD at plan-time (`fe98f893cefed72a56498f3e0271d459c13a1210`),
  recorded URL `git@github.com:deal-lang/spec.git`.
- `tests/showcase` re-pointed to a COMMITTED symlink
  `tests/showcase -> ../spec/examples/showcase` resolving inside the deal
  repo via the submodule mount.
- `scripts/verify-fresh-worktree.sh` + `build.zig phase-1.5-gate-fresh` build
  step — a structural guard rail that creates an ephemeral worktree, runs
  `git submodule update --init --recursive` inside it, and runs `zig build
  phase-1.5-gate` from that ephemeral tree.
- `README.md` newly tracked with the submodule-init instruction.
- `.planning/decisions/ADR-phase-1.5-fresh-worktree-verification.md` locks
  the invariant: **Phase N exit verification MUST be performed from a
  freshly-created git worktree (or ephemeral CI runner) that materializes all
  inputs only via committed git state plus `git submodule update --init
  --recursive`. The main developer checkout is NEVER an acceptable
  verification site.** Future phase gates (phase-2-gate, phase-3-gate, ...)
  MUST add an analogous `-fresh` sibling step.
