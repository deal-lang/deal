## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] parseImportDecl used expect(.ident) rejecting keyword-named package segments**
- **Found during:** Task 3 (oracle parse gate — calcs.deal import failed)
- **Issue:** `import analysis.constraints.{PositiveForce}` failed because `analysis` lexes as `.kw_analysis` (a keyword), but `parseImportDecl` called `expect(.ident)` for path segments. `parseQualifiedNameSegments` already handled this correctly with `isKeyword()`.
- **Fix:** Replaced inline `expect(.ident)` with `if (first.tag == .ident or isKeyword(first.tag))` pattern, matching the existing `parseQualifiedNameSegments` behavior
- **Files modified:** `src/parser_deal.zig`
- **Verification:** `import analysis.constraints.{PositiveForce}` parses correctly; calcs.deal exits 0
- **Committed in:** `0099ac2`

**2. [Rule 1 - Bug] parseConstraintBody infinite loop on STATEMENT_SYNC tokens after error recovery**
- **Found during:** Task 3 (precision.deal caused `deal parse` to hang forever)
- **Issue:** When `syncToStatement` stopped on a `.r_paren` (from `kg(1500)` expression in require statement after a doc comment was consumed by the `else` branch), the `parseConstraintBody` loop would re-enter `else`, emit another E0100, call `syncToStatement` again which would stop immediately — resulting in an infinite loop
- **Fix:** Added forward progress guard after `syncToStatement` (consume semicolon/comma, break on r_brace/eof, force-advance if stuck); also added `.doc_comment` case to handle doc comments inside constraint bodies
- **Files modified:** `src/parser_deal.zig`
- **Verification:** `precision.deal` parses in <1s without hanging; constraints.deal exits 0
- **Committed in:** `0099ac2`

**3. [Rule 1 - Bug] parseCalcBody same forward-progress / doc_comment issues**
- **Found during:** Task 3 (preventative — same structural bug as parseConstraintBody)
- **Issue:** `parseCalcBody` had identical missing forward-progress guard; doc comments inside calc bodies would fall to `else` and potentially cause loops
- **Fix:** Same fix pattern: forward progress guard + `.doc_comment` handling
- **Files modified:** `src/parser_deal.zig`
- **Committed in:** `0099ac2`

**4. [Rule 1 - Bug] comment_attachment.zig, property_span_containment.zig exhaustive switch failures**
- **Found during:** Build verification (zig build test)
- **Issue:** `constraint_def` was removed from the `ElementDef` group but test files still had `constraint_def => |*e| e` in their `elemDef`/`getElementDef` helpers returning wrong type
- **Fix:** Removed `constraint_def` from ElementDef helpers; added 9 new NodeKind arms to property_span_containment.zig visit switch
- **Files modified:** `tests/unit/comment_attachment.zig`, `tests/unit/property_span_containment.zig`
- **Committed in:** `0099ac2`
