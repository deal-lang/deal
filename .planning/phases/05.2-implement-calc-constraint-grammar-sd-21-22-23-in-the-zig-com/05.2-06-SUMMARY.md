## Decisions Made

- **D-07 TextMate heuristic:** `\bsig\b(?=\s+\d)` — match `sig` only when followed by whitespace then a digit. This correctly captures `=> sig 4` and leaves `attribute sig : Real = 1;` uncolored (bare `sig` is followed by ` :` not a digit). Simpler than a lookbehind on `=>` and more robust to intervening whitespace variations.
- **Phase 6 LSP seam:** Wave 5 adds calc/constraint arms to existing semantic-token, hover, and definition handlers. It does NOT refactor the handler infrastructure. Phase 6's broader LSP build-out (workspace-wide index enrichment, cross-file completion, arbitrary-kind hover) extends the SAME arms rather than replacing them. This boundary is documented inline near the new arms in semantic_tokens.rs and hover.rs.
- **go-to-def:** No new code needed for calc/constraint invocation go-to-def. The Index already populates calc_def during definition scan; `pathstring_from_identifier` resolves call callees. A future `pathstring_from_param_decl` helper is the Phase 6 seam point for "jump to parameter" go-to-def.
