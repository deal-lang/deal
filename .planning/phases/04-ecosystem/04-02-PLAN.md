## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| deal.toml → resolver | Untrusted manifest input (git URLs, path strings) controls clone destinations and filesystem reads |
| remote git host → .deal/deps | Cloned repository contents enter the project tree |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-4-02 | Tampering | path dependency in deal.toml (`path = "../../../../etc/passwd"`) | mitigate | Canonicalize + assert_within-style guard in resolver.rs; reject absolute system-path escapes; integration test asserts rejection (V5 input validation) |
| T-4-03 | Spoofing/Tampering | malicious git URL scheme in deal.toml | mitigate | Validate URL scheme is https/ssh/git@/file only before clone; git2 handles transport security |
| T-4-04 | Tampering | non-deterministic deal.lock (HashMap iteration) flaking reproducibility | mitigate | BTreeMap + sort packages by name (D-18); determinism test asserts byte-identical lock across two installs |
| T-4-05 | Information disclosure | vendored repo content trusted by later check | accept | `deal check` only parses (no code execution); vendored tree is the user's policy choice (D-66); documented |
</threat_model>
