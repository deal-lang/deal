## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] m21 fixture form corrected**

- **Found during:** Task 2 execution (first corpus run emitted E0100 not E0123)
- **Issue:** PATTERNS.md specified `@config { voltage: 12, current: 5 }` as the m21 fixture. The bare `@config` annotation is a `standalone annotation` token — `parseStandaloneAnnotation` does NOT call `parseAnnotationBody` when the next token is `{` (it only checks for a `:` value). The `{ ... }` body was left unparsed; the outer parser saw an unexpected `{` and emitted E0100. The E0123 path in the annotation body loop was never reached.
- **Fix:** Changed m21 fixture to use a category annotation form: `@trace:<<satisfies>> R { voltage: 12, current: 5 }`. The `@trace:` prefix produces an `annotation_prefix` token, which routes through `parseCategoryAnnotation`, which calls `parseAnnotationBody` when it sees `{`. This correctly triggers the E0123 path.
- **Files modified:** `tests/malformed/m21_annotation_comma_separator.deal`
- **Commit:** e930993

## D-01 Implementation Detail

The annotation-body loop patch order is:

1. Compute `key_tok = p.peek()` and `key_text` (advance if ident/keyword)
2. NEW: `if (key_tok.tag == .semicolon) { advance; continue; }` — explicit canonical terminator consume
3. `if (key_text.len == 0)` — unknown token: if `.comma` emit E0123; else emit E0100; advance; continue
4. Parse `: value` normally
5. REMOVED: `if (p.peek().tag == .comma) _ = p.advance();` — trailing comma no longer tolerated

The semicolon arm must precede the `key_text.len == 0` check because semicolon produces `key_text = ""` (not ident/keyword), which would otherwise fall into the unknown-token diagnostic path.

## D-02 Implementation Detail

Verification-key switch `else` arm replaced:

```zig
// Before (silent drop):
else => { _ = p.advance(); continue; }

// After (hard diagnostic):
else => {
    try p.diags.append(p.arena, .{
        .code = diagnostics.Codes.e_unknown_verification_key,
        .severity = .err,
        .message = "unknown key in verification block",
        .span = key_tok.span,
    });
    _ = p.advance();
    continue;
}
```

T-05.1-C01 (DoS: infinite loop) mitigated: every arm advances the cursor before `continue`. T-05.1-C02 (silent re-acceptance) mitigated: hard `.err` severity + corpus pin locks first-diagnostic identity.
