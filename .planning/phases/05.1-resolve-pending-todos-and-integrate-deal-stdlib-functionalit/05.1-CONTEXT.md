<decisions>
## Implementation Decisions

### Parser strictness disposition (TODO 1 — I1/I3, the philosophical core)
- **D-01 (I1, annotation-body separator):** `;` is the canonical, **explicitly
  consumed** field terminator (the showcase uses it, e.g. `battery.deal:40-43`). A
  `,` separator is **rejected with a diagnostic** — NOT accepted as an alias. Strict,
  aligned with the Phase 1.5 contract. The parser must stop silently discarding `;`
  (`parser_deal.zig:1914-1923`).
- **D-02 (I3, unknown verification keys):** an unrecognized key inside a verification
  field is a **hard error** (new diagnostic), not silently dropped
  (`parser_deal.zig:1956-1959`) and not a soft warning. Closes the silent-drop
  regression completely.
- **D-03 (I2, keyword-as-field-key):** keyword field keys (e.g. `from:` in
  `behaviors.deal:29`) must be accepted because the showcase oracle relies on them —
  but as an **explicit, tested accept** via `AnnotationKey ::= IDENT | _Keyword`, not
  a silent one. Lexer stays flat-reserved; no contextual machinery.

### Array unification (TODO 1 — I4)
- **D-04:** Introduce a **real `ArrayLiteral` AST node** and thread it through the
  full pipeline (expr.zig → AST → IR/lowering → codegen). Add `ArrayLiteral` to
  `PrimaryExpression` as the single source of truth in `expr.zig`; **delete both**
  duplicated parsers (`parser_dealx.parseArrayLiteral:809`,
  `parser_deal.parseVerificationArray:1994`). Replace the synthetic
  `call(__array__, items)` node. Acceptance requires **byte-identical output** vs.
  pre-refactor goldens in both array contexts (regenerate IR/SysML goldens as needed).
  Keep `__array__` only if codegen back-compat genuinely forces it (verify in research).

### gap block (TODO 1 — I5)
- **D-05:** **Honest-opaque.** Redefine `GapBlock` in `dealx.ebnf` to match the
  parser's actual opaque brace-balanced byte-scan reality
  (`parser_dealx.zig:1230-1247`) — stop the spec claiming structure the parser
  doesn't deliver. Structured-gap field parsing is **deferred** (Wave E / future
  phase, pairs with `deal check --verify` consuming gap data).

### Spec submodule handling (TODO 1 — Wave A)
